## 一、执行摘要

经过对 Step-RL v2.0 全系统（约 4,500+ 行核心代码、23 个 Python 模块、13 个脚本）的深度代码审查与架构分析，本报告从 **9 大维度** 识别出 **31 个具体优化方向**，其中：

| 优先级 | 数量 | 预估综合收益 |

| 🔴 P0（即刻实施） | 8 项 | 完成率 +3~5pp，训练速度 +40% |

**核心洞察**：当前系统的瓶颈已不在"奖励设计"本身，而在于 **(1) 样本效率天花板**（on-policy 采样成本高）、**(2) 推理架构效率**（单次前向传播未充分利用）、**(3) 状态表示粒度**（纯文本 DOM 丢失视觉空间信息）。解决这三个根因，有望将系统推入"95% 完成率俱乐部"。

**现状**：当前 PPO/GRPO 为自定义简化实现，README 已明确标注建议迁移。

### 2.2 实现优先经验回放（PER）（P0）

**现状**：当前 Replay Buffer 为 uniform sampling 的 `deque`（`history_ratio=0.25` 混入历史轨迹），未实现真正的优先采样。

**问题**：
- 高回报与失败轨迹被同等采样，样本效率低
- 配置中 `alpha_prioritized=0.6` 和 `beta_prioritized=0.4` 已定义但**未在代码中使用**

**优化方案**：
```python
import numpy as np
from collections import deque
import random

class PrioritizedReplayBuffer:
    """Sum-Tree based PER for trajectory-level sampling."""
    
    def __init__(self, capacity: int = 10000, alpha: float = 0.6, beta: float = 0.4):
        self.capacity = capacity
        self.alpha = alpha  # priority exponent
        self.beta = beta    # importance sampling annealing
        self.tree = SumTree(capacity)  # 需要实现 SumTree
        self._trajectories = {}
        self._max_priority = 1.0
    
    def add(self, trajectory: Trajectory, priority: Optional[float] = None):
        """Add with max priority (ensures new trajectories get sampled)."""
        p = priority if priority is not None else self._max_priority
        p = (abs(p) + 1e-6) ** self.alpha
        idx = self.tree.add(p)
        self._trajectories[idx] = trajectory
    
    def sample(self, batch_size: int) -> Tuple[List[Trajectory], np.ndarray, np.ndarray]:
        """Sample with importance sampling weights."""
        indices, priorities = [], []
        for _ in range(batch_size):
            s = random.random() * self.tree.total()
            idx, p = self.tree.get(s)
            indices.append(idx)
            priorities.append(p)
        
        # Importance sampling weights
        probs = np.array(priorities) / self.tree.total()
        weights = (len(self._trajectories) * probs) ** (-self.beta)
        weights /= weights.max()  # normalize
        
        trajectories = [self._trajectories[idx] for idx in indices]
        return trajectories, indices, weights
```

**关键设计**：
- 优先度基于 `|return - baseline|`（TD 误差代理），同时保留高回报和失败轨迹
- 在 `base_trainer.py` 的 `collect_rollouts()` 后，用 `Trajectory.total_return` 作为初始优先级
- 更新时根据新计算的 advantage 调整优先级

**预期收益**：样本效率提升 20~30%，收敛步数减少 15%。

### 2.3 引入 GAE 自适应 λ（P1）

**现状**：GAE 固定 `λ=0.95`，不区分任务阶段。

**问题**：
- 早期训练（高探索）：需要低 λ 降低方差，增强稳定性
- 后期训练（高利用）：需要高 λ 减少 bias，精确信用分配
- 长链路任务（>15 步）：固定 λ 导致尾部梯度衰减严重

**优化方案**：
```python
def adaptive_gae_lambda(epoch: int, total_epochs: int, 
                        avg_episode_length: float) -> float:
    """Dynamic λ scheduling based on training stage and trajectory length."""
    progress = epoch / total_epochs
    
    # Base: early low, late high
    base = 0.85 + 0.15 * progress  # [0.85, 1.0]
    
    # Adjust for trajectory length: longer tasks need higher λ
    # to reduce bias in credit assignment
    length_factor = min(0.1, 0.02 * (avg_episode_length - 10))
    
    return min(0.99, base + length_factor)
```

**预期收益**：长链路任务（Level 3/4）收敛速度 +15%。

## 三、奖励塑形与信用分配优化

### 3.1 动态 KL 自适应（P0）

**现状**：KL 系数固定（`kl_coef=0.1`），或仅实现简单自适应（`kl_adaptive=true`）。

**问题**：
- 训练初期：策略与参考模型差异大，高 KL 限制探索
- 训练后期：策略已收敛，低 KL 可能不足防止偏离
- 未考虑不同任务的 KL 需求差异

**优化方案**：实现 `AdaptiveKLController`（类似 OpenAI InstructGPT 方案）：
```python
class AdaptiveKLController:
    """PID-like adaptive KL coefficient controller."""
    
    def __init__(self, init_kl_coef: float = 0.1, target_kl: float = 0.1,
                 kl_horizon: int = 10000):
        self.kl_coef = init_kl_coef
        self.target_kl = target_kl
        self.kl_horizon = kl_horizon
        self._kl_history = deque(maxlen=100)
    
    def update(self, current_kl: float) -> float:
        self._kl_history.append(current_kl)
        avg_kl = np.mean(self._kl_history)
        
        # Proportional adjustment
        if avg_kl > self.target_kl * 1.5:
            self.kl_coef *= 1.05  # 增加惩罚
        elif avg_kl < self.target_kl * 0.5:
            self.kl_coef *= 0.95  # 减少惩罚
        
        self.kl_coef = np.clip(self.kl_coef, 0.01, 0.5)
        return self.kl_coef
```

**预期收益**：训练稳定性 +25%，减少策略崩溃。

### 3.2 课程调度器智能化升级（P1）

**现状**：课程调度使用简单线性插值（`early`→`late`），晋升阈值固定（90%）。

**问题**：
- 未根据实时训练指标动态调整（如 loss 曲线、KL 值）
- 固定阈值不适用于不同任务类型
- 没有"降级"机制，一旦晋升即使失败率上升也无法回退

**优化方案**：实现 **Bandit-based 课程调度**：
```python
class BanditCurriculumScheduler(CurriculumScheduler):
    """Use Multi-Armed Bandit to dynamically select task difficulty."""
    
    def __init__(self, *args, epsilon: float = 0.1, **kwargs):
        super().__init__(*args, **kwargs)
        self.epsilon = epsilon
        # UCB score tracking
        self._level_rewards = {lvl: [] for lvl in self.levels}
        self._level_counts = {lvl: 0 for lvl in self.levels}
    
    def sample_task(self, epoch: int) -> Optional[Task]:
        # ε-greedy: exploit best level or explore random
        if random.random() < self.epsilon:
            return self._explore_task()
        
        # UCB1: select level with highest upper confidence bound
        best_level = None
        best_ucb = -float('inf')
        for lvl in self.levels:
            if lvl > self._current_level + 1:  # 不能跳级太多
                continue
            n = self._level_counts[lvl]
            if n == 0:
                return self._sample_from_level(lvl)
            avg_reward = np.mean(self._level_rewards[lvl])
            ucb = avg_reward + np.sqrt(2 * np.log(sum(self._level_counts.values())) / n)
            if ucb > best_ucb:
                best_ucb = ucb
                best_level = lvl
        
        return self._sample_from_level(best_level)
```

**预期收益**：课程学习效率 +20%，避免在过难任务上浪费样本。

### 3.3 分步进度奖励替代全局进度差（P1）

**现状**：`r_progress = progress(t) - progress(t-1)`，即全局进度的差值。

**问题**：
- 如果某一步操作是"必要但表面上无进展"（如等待页面加载），`r_progress` 可能为 0 或负
- 不同任务路径的进度曲线形状差异大，导致奖励不可比

**优化方案**：引入 **局部子目标完成度（Sub-goal Completion）**：
```python
def compute_subgoal_progress(task: Task, action_history: List, obs: Observation) -> float:
    """Decompose task into sub-goals and reward sub-goal completion."""
    subgoals = task.subgoals  # e.g., ["search", "select", "checkout", "confirm"]
    
    # 使用 LLM-as-Judge 或规则匹配判断当前完成到哪个子目标
    current_subgoal_idx = estimate_subgoal_index(action_history, obs)
    
    # 奖励子目标完成而非全局进度
    if current_subgoal_idx > previous_subgoal_idx:
        return 0.3  # 子目标完成奖励
    elif progress > 0:  # 子目标内进展
        return progress * 0.1
    else:
        return 0.0
```

**实施路径**：
1. 在任务定义中显式标注子目标（或 LLM 自动分解）
2. 在 `ProgressEstimator` 中加入 `subgoal_head` 并行预测子目标
3. 总进度 = 子目标进度 × 0.7 + 细粒度进度 × 0.3

**预期收益**：长链路任务完成率 +2~3pp。

### 3.4 基于不确定性的奖励裁剪（P1）

**现状**：不确定性仅用于 `r_progress_weighted = r_progress * uncertainty_penalty`，但未处理极端情况。

**优化方案**：当 `uncertainty > 0.7` 时，不仅降低权重，还应：
1. 跳过该样本的 PPO 更新（防止噪声信号误导）
2. 将该状态加入"高不确定性待标注队列"，供人工审核
3. 在 Progress Estimator 中对该样本施加更大的监督权重

```python
if uncertainty > 0.7:

    info["skipped_update"] = True
    info["high_uncertainty"] = True
```

```python
from transformers import Qwen2VLForConditionalGeneration, AutoProcessor

class MultimodalObservationEncoder:
    """Encode both text DOM and screenshot."""
    
    def __init__(self, text_encoder, vision_encoder):
        self.text_encoder = text_encoder  # Qwen3-8B
        self.vision_encoder = vision_encoder  # Qwen2-VL-7B or CLIP
        self.fusion_layer = nn.Linear(
            text_encoder.config.hidden_size + vision_encoder.config.hidden_size,
            text_encoder.config.hidden_size
        )
    
    def encode(self, obs_text: str, screenshot_b64: str) -> torch.Tensor:
        # Text encoding
        text_emb = self.text_encoder(**tokenize(obs_text)).last_hidden_state[:, -1]
```

### 4.2 MinHash 性能重构（P0）

**现状**：MinHash 已优化（预计算排列替代 MD5 循环），但仍使用 `hashlib.md5(s.encode())` 对每个 shingle 进行哈希。

**问题**：
- 每次状态哈希需要 `num_perm * num_shingles` 次 MD5 调用，虽然比原版 64,000 次 MD5 好很多，但仍有优化空间
- 无 SIMD 加速，CPU 密集型

**优化方案**：

**方案 A：使用 xxHash 替代 MD5**
```python
import xxhash  # pip install xxhash

def _minhash_fast(self, text: str, url: str, num_perm: int = 64) -> str:
    words = text.lower().split()
    if len(words) < 2:
        return self._simple_hash(text, url)
    
    # 使用 xxHash（比 MD5 快 5~10x）
    shingles = {f"{words[i]} {words[i+1]}" for i in range(len(words) - 1)}
    
    p = (1 << 61) - 1
    hashes = []
    for seed in range(num_perm):
        a = (seed * 1234567891 + 1) & 0xFFFFFFFFFFFFFFFF
        b = (seed * 9876543211 + 1) & 0xFFFFFFFFFFFFFFFF
        min_val = p
        for s in shingles:
            h = xxhash.xxh64(s, seed=seed).intdigest()  # 64-bit fast hash
            perm = ((h * a + b) & 0xFFFFFFFFFFFFFFFF) % p
            if perm < min_val:
                min_val = perm
        hashes.append(min_val)
    # ... 余同
```

**方案 B：使用 SimHash 替代 MinHash**（适合高维文本）
- SimHash 是局部敏感哈希的另一种实现，对文本相似度更鲁棒
- 汉明距离计算比 MinHash 的 Jaccard 估计更快

**方案 C：缓存状态哈希**（最简单有效）
```python
class StateMemory:
    def __init__(self, *args, **kwargs):
        super().__init__(*args, **kwargs)
        self._hash_cache = {}  # LRU cache for hash results
    
    def compute_hash(self, observation_text: str, url: str, *args):
        cache_key = hashlib.md5(f"{url}|{observation_text[:200]}".encode()).hexdigest()
        if cache_key in self._hash_cache:
            return self._hash_cache[cache_key]
        h = super().compute_hash(observation_text, url, *args)
        self._hash_cache[cache_key] = h
        return h
```

**预期收益**：状态哈希速度 +3~5x，环境交互延迟降低。

### 4.3 分层状态记忆（短期+长期）（P2）

**现状**：单一 `OrderedDict` 存储所有访问状态，没有区分短期和长期记忆。

**优化方案**：实现双轨记忆系统：
```python
class HierarchicalStateMemory:
    """Short-term episodic memory + Long-term semantic memory."""
    
    def __init__(self):
        # Short-term: 当前 episode 内的状态序列（检测局部循环）
        self.episodic_buffer = deque(maxlen=200)
        
        # Long-term: 跨 episode 的语义状态（识别相似任务）
        self.semantic_memory = {}  # 存储状态-动作价值对
        
        # 状态抽象层：将具体 DOM 抽象为语义模板
        self.abstraction_layer = StateAbstraction()
    
    def update(self, state_hash: str, obs_text: str, action: str, reward: float):
        # 短期记忆更新（循环检测）
        self.episodic_buffer.append(state_hash)
        
        # 抽象化并存储到长期记忆
        abstract_state = self.abstraction_layer.abstract(obs_text)
        key = f"{abstract_state}|{action}"
        
        # 更新状态-动作价值（类似 Q-learning）
        if key not in self.semantic_memory:
            self.semantic_memory[key] = {"q_value": 0.0, "count": 0}
        
        mem = self.semantic_memory[key]
        mem["count"] += 1
        mem["q_value"] += 0.1 * (reward - mem["q_value"])  # TD update
```

**状态抽象示例**：
- 将 `https://example.com/products/12345` 抽象为 `product_detail_page`
- 将 `<button data-testid="buy-now-12345">` 抽象为 `buy_button`
- 这样跨网站、跨产品的相似状态可以共享价值估计

**预期收益**：跨任务迁移能力，新任务样本效率 +30%。

### 4.4 基于 Transformer 的轨迹表示（P2）

**现状**：状态记忆仅存储哈希值，没有学习状态表示。

**优化方向**：训练一个轻量的 Transformer Encoder 来学习状态表示：
```python
class TrajectoryEncoder(nn.Module):
    """Encode a trajectory into a fixed-length vector for better credit assignment."""
    
    def __init__(self, state_dim: int, hidden_dim: int = 256):
        super().__init__()
        self.encoder = nn.TransformerEncoder(
            nn.TransformerEncoderLayer(d_model=state_dim, nhead=8, dim_feedforward=hidden_dim),
            num_layers=2
        )
        self.pooling = nn.AdaptiveAvgPool1d(1)
    
    def forward(self, state_sequence: List[torch.Tensor]) -> torch.Tensor:
        # state_sequence: [seq_len, state_dim]
        x = torch.stack(state_sequences).unsqueeze(1)  # [seq_len, 1, state_dim]
        encoded = self.encoder(x)  # [seq_len, 1, state_dim]
        return encoded.squeeze(1)  # [seq_len, state_dim] - per-step context-aware embedding
```

**用途**：
- 替代简单的状态哈希，用学习的表示进行循环检测
- 更精确地识别语义相似但表面不同的状态
- 为 Progress Estimator 提供上下文增强的输入

**预期收益**：循环检测误报率降低 50%。

### 5.1 智能等待与 SPA 检测（P0）

**现状**：`spa_wait_ms=1000` 固定等待，未检测实际渲染状态。

**问题**：

```python
async def smart_wait(page: Page, action_params: Dict, max_wait_ms: int = 5000):
    """Adaptive wait with SPA render detection."""
    start_time = time.time()
    
    # 1. 检查目标元素是否已出现（最快路径）
    locator = page.locator(f"[{action_params['element_id']}]")
    try:
        await locator.wait_for(state="visible", timeout=2000)
        return True, 0  # 无需等待
    except:
        pass
    
    # 2. 检测 SPA 渲染状态
    spa_ready = await page.evaluate("""() => {
        // Vue 3
        if (window.__VUE__ && document.querySelector('[data-v-app]')) return 'vue';
        // React
        if (window.__REACT__ || document.querySelector('[data-reactroot]')) return 'react';
        // 通用：DOM 稳定检测
        return new Promise(resolve => {
            let lastHTML = document.body.innerHTML;
            let stable_count = 0;
            const check = () => {
                if (document.body.innerHTML === lastHTML) {
                    stable_count++;
                    if (stable_count >= 3) return resolve('stable');
                } else {
                    stable_count = 0;
                    lastHTML = document.body.innerHTML;
                }
                setTimeout(check, 100);
            };
            check();
        });
    }""")
    
    # 3. 如果 SPA 已就绪，再检查元素
    if spa_ready:
        try:
            await locator.wait_for(state="visible", timeout=max_wait_ms - 2000)
            return True, elapsed_ms
        except:
            return False, elapsed_ms
```

**预期收益**：平均等待时间从 1000ms 降至 200~300ms，任务完成时长 -15%。

### 5.2 Grounding 结果缓存（P1）

**现状**：每次动作都重新查询 DOM 验证元素，即使页面未变化。

**优化方案**：缓存同一页面状态下的 Grounding 结果：
```python
class GroundingCache:
    """Cache element validation results per page state."""
    
    def __init__(self, ttl_seconds: float = 5.0):
        self.cache = {}
        self.ttl = ttl_seconds
    
    def _key(self, page_url: str, action_params: Dict) -> str:
        return hashlib.md5(f"{page_url}|{json.dumps(action_params, sort_keys=True)}".encode()).hexdigest()
    
    async def validate_with_cache(self, page: Page, action: str, params: Dict):
        key = self._key(page.url, params)
        if key in self.cache:
            result, timestamp = self.cache[key]
            if time.time() - timestamp < self.ttl:
                return result
        
        result = await self._actual_validate(page, action, params)
        self.cache[key] = (result, time.time())
        return result
    
    def invalidate(self, url: str):
        """Invalidate all entries for a URL when page changes."""
        self.cache = {k: v for k, v in self.cache.items() if not v[0].startswith(url)}
```

**预期收益**：同一页面多动作执行时，Grounding 延迟 -50%。

```python
class ElementFingerprintDB:
    """Learn robust selectors from successful trajectories."""
    
    def __init__(self):
        self.fingerprints = {}  # {domain: {element_role: [successful_patterns]}}
    
    def record_success(self, url: str, action_params: Dict, success: bool):
        domain = extract_domain(url)
        role = action_params.get("action")
        
        if success:
            pattern = self._extract_pattern(action_params)
            self.fingerprints.setdefault(domain, {}).setdefault(role, []).append(pattern)
    
    def suggest_selector(self, page: Page, action: str, target_text: str) -> Optional[str]:
        """Suggest a selector based on historical success patterns."""
        domain = extract_domain(page.url)
        patterns = self.fingerprints.get(domain, {}).get(action, [])
        
        # 选择成功率最高的模式
        best_pattern = max(set(patterns), key=patterns.count, default=None)
        return best_pattern
```

```python
# 蒸馏损失
def distillation_loss(teacher_out, student_out, temperature: float = 2.0):
    # MSE on progress
    mse = F.mse_loss(student_out.progress, teacher_out.progress)
    
    # KL on uncertainty distribution
    teacher_unc = teacher_out.uncertainty
    student_unc = student_out.uncertainty
    kl = F.kl_div(
        F.log_softmax(student_unc / temperature, dim=-1),
        F.softmax(teacher_unc / temperature, dim=-1),
        reduction="batchmean"
    ) * (temperature ** 2)
    
    return mse + 0.5 * kl
```

```python
        }, temp_path)
        os.replace(temp_path, path)  # atomic rename
    
    def load(self, trainer, path: str) -> bool:
        """Load with validation."""
        if not os.path.exists(path):
            return False
        try:
            ckpt = torch.load(path, weights_only=True)
            # Validate checkpoint integrity
            assert "policy_state" in ckpt
            assert "global_step" in ckpt
            trainer.policy.load_state_dict(ckpt["policy_state"])
            trainer.policy_optimizer.load_state_dict(ckpt["optimizer_state"])
            trainer.global_step = ckpt["global_step"]
            trainer.epoch = ckpt["epoch"]
            return True
        except Exception as e:
            logger.error(f"Checkpoint corrupt: {e}. Starting from scratch.")
            return False
```
